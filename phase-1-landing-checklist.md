# Phase 1 Landing Checklist - PERUMDA TIRTAWENING

Dokumen ini adalah checklist harian implementasi **Landing Page v1** sebagai fase awal produk.

## Target Phase

- Membangun landing page publik yang profesional, responsif, dan siap dipakai sebagai wajah resmi informasi digital.
- Menyediakan fondasi konten yang mudah dikembangkan ke fase dashboard internal.

## Durasi

- Estimasi: **10 hari kerja** (2 minggu).

## Definisi Selesai (Definition of Done)

- Landing page publik dapat diakses stabil di desktop dan mobile.
- Semua section utama terisi konten versi awal yang jelas.
- Navigasi CTA berfungsi.
- Performa dan aksesibilitas memenuhi baseline internal.

---

## Checklist Harian Implementasi

## Hari 1 - Discovery Konten dan Struktur Informasi

- [ ] Validasi objective landing page (informasi publik, trust, CTA).
- [ ] Tentukan sitemap landing v1 (Home, Layanan, FAQ, Kontak).
- [ ] Kumpulkan copy awal per section (headline, subheadline, value proposition).
- [ ] Tentukan data statistik publik yang boleh ditampilkan.

Output:

- Draft konten awal dan struktur halaman final.

## Hari 2 - Fondasi UI dan Branding

- [ ] Tetapkan identitas visual (warna utama, warna netral, tone resmi BUMD).
- [ ] Definisikan token dasar UI (spacing, radius, heading scale).
- [ ] Finalisasi style guideline komponen landing.
- [ ] Setup komponen reusable awal (section wrapper, card, CTA button).

Output:

- Baseline design system ringan untuk landing page.

## Hari 3 - Implementasi Header, Hero, dan CTA Utama

- [ ] Implement sticky header + navigasi anchor section.
- [ ] Implement hero section (headline, subheadline, 2 CTA).
- [ ] Tambahkan badge/identitas institusi.
- [ ] Pastikan responsive di viewport kecil.

Output:

- Header + Hero production-ready.

## Hari 4 - Implementasi Statistik dan Problem-Solution

- [ ] Bangun section statistik ringkas (cakupan, titik data, unit KPI).
- [ ] Bangun section problem-solution operasional.
- [ ] Tambahkan visual hierarchy yang jelas (judul, deskripsi, highlight).

Output:

- Section statistik dan problem-solution aktif.

## Hari 5 - Implementasi Section Fitur Layanan

- [ ] Tampilkan 4 unit utama (Distribusi, Pencatat Meter, Langganan, RPM).
- [ ] Tambahkan deskripsi manfaat tiap unit.
- [ ] Tambahkan quick-link ke dashboard internal (jika relevan).

Output:

- Section fitur layanan modul lengkap.

## Hari 6 - Implementasi Transparansi, KPI, dan Roadmap Layanan

- [ ] Tambahkan section KPI operasional tingkat tinggi.
- [ ] Tambahkan section roadmap layanan digital (fase pengembangan).
- [ ] Tambahkan callout untuk inisiatif GIS dan pengendalian NRW.

Output:

- Section transparansi dan roadmap tersedia.

## Hari 7 - Implementasi FAQ dan Kontak Resmi

- [ ] Buat FAQ inti pelanggan (layanan, cakupan, kanal komunikasi).
- [ ] Tambahkan kontak resmi (alamat, call center, email, jam layanan).
- [ ] Tambahkan CTA penutup yang jelas.

Output:

- FAQ + kontak publik siap.

## Hari 8 - Optimasi SEO dan Metadata

- [ ] Susun title dan meta description yang sesuai.
- [ ] Tambahkan Open Graph dasar.
- [ ] Pastikan heading semantics (h1-h2-h3) benar.
- [ ] Periksa internal linking.

Output:

- Baseline SEO teknis v1.

## Hari 9 - Quality Check UI/UX

- [ ] Uji tampilan lintas ukuran layar.
- [ ] Lengkapi state hover/focus/active untuk interaksi penting.
- [ ] Rapikan spacing dan konsistensi tipografi.
- [ ] Validasi aksesibilitas dasar (kontras warna, keyboard nav dasar).

Output:

- Landing page siap review stakeholder.

## Hari 10 - Finalisasi dan Handover Phase 1

- [ ] Bug fixing prioritas tinggi.
- [ ] Final review konten dan legal wording.
- [ ] Dokumentasikan perubahan dan daftar backlog phase berikutnya.
- [ ] Demo ke stakeholder dan kumpulkan feedback.

Output:

- Landing page v1 dirilis untuk tahap awal.

---

## Backlog Lanjutan (Setelah Phase 1 Selesai)

- [ ] Integrasi data statistik dinamis dari API.
- [ ] Form kontak terhubung ke kanal internal.
- [ ] Penambahan halaman detail layanan.
- [ ] Multi-bahasa (jika dibutuhkan).
